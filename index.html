Workshop premise

AI tools are great. Random terminal bravery is not.

This page is intentionally silly, but the workshop behind it should be strict about safe defaults: check the docs first, understand the workflow you are invoking, work in the right environment, and avoid commands that mutate systems unless you know the blast radius.

Danger Zone

Commands that can ruin your day fast.

These are not "cool power user" shortcuts. They are examples of commands that can delete files, rewrite history, damage disks, leak credentials, or mutate the wrong environment if someone copies them blindly from an AI answer.

File deletion

rm -rf

Recursively deletes files and folders without asking. Run in the wrong place and you can wipe a project or much more.

Disk overwrite

dd if=... of=/dev/diskX

Writes raw data directly to a disk device. One wrong target can destroy a drive or partition table.

Filesystem format

mkfs / diskutil eraseDisk

Formats a volume. Useful on purpose, catastrophic when aimed at the wrong disk.

Ownership and permissions

chmod -R / chown -R

Changes access rules across entire directories. Bad recursion can lock users out or expose sensitive files.

Git history rewrite

git reset --hard

Throws away local tracked changes and resets the checkout. Great for accidents, terrible for uncommitted work.

Untracked file purge

git clean -fd

Deletes untracked files and folders. Easy way to remove generated junk, easy way to remove valuable local files too.

Remote overwrite

git push --force

Rewrites branch history on the remote. Used carelessly, it can erase teammates' commits from shared branches.

Secret exposure

env, printenv, cat .env

These can dump tokens, API keys, passwords, and internal endpoints into logs, screenshots, or AI chat transcripts.
